# Яндекс Маршруты — Итоговый проект 2 спринта

## Описание

Ручное тестирование веб‑сервиса **Яндекс Маршруты**: тест-анализ, классы эквивалентности и граничные значения, тест-кейсы и баг-репорты.

Цель работы — оценить качество реализации валидации полей времени и адресов, а также корректность расчёта стоимости и времени поездки на собственном автомобиле. На основе результатов сформированы выводы о готовности продукта к релизу.

## Что демонстрирует этот проект

- **Тест-анализ и работа с требованиями**
- **Проектирование тест-кейсов на основе КЭ и ГЗ**
- **Оформление баг-репортов (с приоритами и Severity)**
- **Оценку качества продукта и формирование рекомендаций по релизу**

## Структура репозитория

- **analysis/** — общий тест-анализ сервиса: ключевые пользовательские сценарии, риски и выводы по качеству продукта.
- **ce-gz/** — классы эквивалентности и граничные значения для:
  - полей времени (**Часы**, **Минуты**),
  - полей адресов (**Откуда**, **Куда**),
  - маршрутов и интервалов времени.
- **testcases/** — тест-кейсы, сгруппированные по областям:
  - [`TC-hours.md`](testcases/TC-hours.md) — валидация поля **Часы** (`T1–T24`)
  - [`TC-minutes.md`](testcases/TC-minutes.md) — валидация поля **Минуты** (`T25–T48`)
  - [`TC-from-address.md`](testcases/TC-from-address.md) — поле **Откуда** (`T49–T53`)
  - [`TC-to-address.md`](testcases/TC-to-address.md) — поле **Куда** (`T55–T59`)
  - [`TC-own-car-cost.md`](testcases/TC-own-car-cost.md) — расчёт стоимости/времени на своём авто (`T60–T65`)
- **bugreports/** — баг-репорты по найденным дефектам, в том числе критическим (`BUG2`, `BUG3`, `BUG4`, `BUG6`, `BUG7`, `BUG11`, `BUG14`, `BUG15`, `BUG16`).

## Трассируемость: Требования → Тест-кейсы → Баги

Для критичных требований к валидации полей и расчёту маршрутов построена простая матрица трассируемости (пример, подстрой под свою фактическую нумерацию):

| Требование                                        | Тест-кейсы                         | Найденные баги                    |
|---------------------------------------------------|------------------------------------|-----------------------------------|
| RQ-1: Валидация поля «Часы»                      | T1–T24 (`TC-hours.md`)             | BUG2, BUG3, BUG4                  |
| RQ-2: Валидация поля «Минуты»                    | T25–T48 (`TC-minutes.md`)          | BUG6, BUG7                        |
| RQ-3: Валидация адресов «Откуда» / «Куда»        | T49–T53, T55–T59 (`TC-from/to`)    | BUG11, BUG14, BUG15, BUG16        |
| RQ-4: Расчёт стоимости и времени на своём авто   | T60–T65 (`TC-own-car-cost.md`)     | (см. соответствующие баги)        |

Каждый тест-кейс оформлен по единому шаблону:

- **ID** (например, `T25`)
- **Название**
- **Предусловия**
- **Шаги**
- **Ожидаемый результат**
- **Тип** (позитивный / негативный)
- **Приоритет**

Это облегчает поддержку набора тестов, повторное использование сценариев и последующую автоматизацию.
