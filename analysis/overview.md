# Тест-анализ сервиса Яндекс Маршруты

Вкладка Google Sheets: «Этап 1. Тест-анализ».

## Объекты тестирования

- **Поле ввода Часы**
  - Формат: принимаются только числовые значения, отображение в двухразрядном формате `HH`.
  - Диапазон допустимых значений: целые числа от 0 до 23 включительно.
  - Автоматическая подстановка нуля: при вводе однозначного числа (0–9) система сама добавляет ведущий ноль (9 → 09).
  - Обработка некорректного ввода чисел: значения <0, >23, дробные числа, слишком большие числа, пустое значение при необходимости ввода.
  - Обработка некорректного ввода символов: буквы, спецсимволы, пробелы, смешанный ввод (цифры + буквы).
  - Отображение ошибки: при некорректном вводе поле подсвечивается красным.
  - Текст ошибки: «Вы ввели некорректное время».
  - Сброс ошибки: подсветка и сообщение исчезают после ввода корректного значения.

- **Поле ввода Минуты**
  - Формат: только числовые значения, отображение в формате `MM`.
  - Диапазон допустимых значений: целые числа от 0 до 59 включительно.
  - Автоматическая подстановка нуля для однозначных чисел (0–9).
  - Обработка некорректных числовых и символьных значений аналогична полю Часы.

- **Поля ввода адресов Откуда и Куда**
  - Допустимы только адреса из списка требований:
    - Усачева, 3
    - Комсомольский проспект, 18
    - Зубовский бульвар, 37
    - М. Пироговская, 25
    - Хамовнический Вал, 34
    - Фрунзенская набережная, 46
    - 3-я Фрунзенская улица, 12
  - Обработка адресов, не входящих в список.
  - Обработка пробелов до и после адреса.
  - Обязательность полей (пустое/заполненное значение).
  - Отображение и текст ошибок при некорректном вводе.

- **Расчёт маршрута и стоимости**
  - Варианты расстояния:
    - Расстояние = 0 (точка А совпадает с точкой B).
    - Расстояние > 0 (разные адреса из списка).
  - Временные интервалы начала поездки:
    - 00:01–08:00 — ночь.
    - 08:01–12:00 — утро.
    - 12:01–18:00 — день.
    - 18:01–22:00 — вечер.
    - 22:01–00:00 — поздний вечер.
  - Тип поездки: «Свой» → «Автомобиль».
  - Проверка корректности:
    - стоимости поездки (рубли),
    - времени в пути (минуты) для разных интервалов и расстояний.

## Основные риски

- Некорректная обработка граничных значений для времени:
  - 0 и 23/24 для часов,
  - 0 и 59/60 для минут.
- Неправильная работа с пробелами и невалидными символами в полях ввода.
- Отсутствие ошибок при пустых обязательных полях Часы, Минуты, Откуда, Куда.
- Ошибки в формулах расчёта стоимости и времени поездки при расстоянии > 0.

## Вывод

Тест-анализ показывает необходимость тщательного покрытия:

- валидации полей Часы и Минуты (включая негативные сценарии и границы диапазонов),
- валидации полей Откуда и Куда (валидные/невалидные адреса, пробелы, обязательность),
- расчёта стоимости и времени поездки на собственном автомобиле для всех временных интервалов и вариантов расстояния.

Найденные риски подтверждаются тест-кейсами и баг-репортами во вкладках «Этап 3. Тест-кейсы» и «Этап 4. Баг-репорты».
